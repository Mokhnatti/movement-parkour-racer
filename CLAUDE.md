# CLAUDE.md — Movement Parkour Racer (проект №1)

> Полный концепт и план: [PLAN.md](./PLAN.md). Этот файл — рабочие правила для Claude по проекту.
> Скоростная паркур-гонка от 3 лица, киберпанк, Unreal Engine 5.8.

## ГЛАВНЫЙ ПРИНЦИП
**80% успеха = game feel.** Лучшее — враг хорошего. Сначала фил на серой болванке, потом всё остальное. Скорость должна быть ЗАРАБОТАНА (накопление импульса за чистоту), не выдана.
Главная задача всегда: **гонять и крутить параметры, пока не вкусно.**

## ЖЕЛЕЗНЫЙ ПОРЯДОК РАБОТЫ (не перепрыгивать)
1. Фил — манекен в пустой коробке + камера.
2. Одна ручная трасса + таймер + рестарт + свой призрак.
3. Призраки + лидерборд (асинхрон).
4. Правила трасс → модульная библиотека → дейлики (генерация).
5. Тюнинг статов (опц.).
6. Realtime-сеть — только в самом конце, если вообще.
Каждый этап дешевле провалить, чем следующий. Не лезть вперёд.

## ТВЁРДЫЕ РЕШЕНИЯ (не пересматривать без причины)
- **Камера: третье лицо** (на болванке проверить обе, решить руками).
- **Движок: UE 5.8, НЕ UE6.** UE6 ломает Blueprints/Actors, Verse-основной, сырой до 2027+. Большие карты решены в 5.8 через World Partition.
- **Фиксированный таймстеп физики** (не к фрейму) — заложить СРАЗУ, иначе лидерборды нечестные на разном FPS.
- **Соревнование — асинхрон (призраки + лидерборды + daily по сиду)**, НЕ realtime-неткод (смерть для малых команд).
- **Параметры мувмента — наружу** (DataAsset/BP), менять на лету. Быстрый цикл правка→проверка — главный фактор.

## НЕ ДЕЛАТЬ НА БОЛВАНКЕ (антискоуп)
Графику, трассы, Lumen/Nanite (выключить — 300+ FPS), бой, меню, тюнинг статов, свою модель, «идеальную песочницу параметров» месяц. Только фил.

## GAME FEEL — ОБЯЗАТЕЛЬНЫЙ МИНИМУМ
- Прощающие окна: coyote time (~5 кадров), jump buffering, corner correction, halved-gravity на пике прыжка.
- Камера продаёт скорость: FOV растёт со скоростью (главный трюк), наклон на вал-ране/поворотах, motion blur на пике, микро-тряска.
- Ядро: momentum preservation, накопление за чистоту (видимый индикатор), сброс при столкновении, momentum storage с платформ.
- Мувсет: бег, прыжок, рывок (КД), вал-ран, слайд с поджатием.

## ЛЕВЕЛДИЗАЙН
«Широкий коридор» А→Б, внутри 3 линии (безопасная/быстрая/безумная) с РАЗНЫМ временем. НЕ полная песочница (ломает «правильную линию» и лидерборд). Геометрия трассы = геймплей; ассеты — последний слой (whitebox → кайфово бежать → одеваем).

## DATA-DRIVEN ТРАССЫ
Фил → цифры → правила (код, единый источник истины) → 10–15 модулей РУКАМИ → потом генератор (WFC, не рандом) → текстуры. Генератор НЕ раньше времени.

## ИНТЕГРАЦИЯ CLAUDE / MCP (UE 5.8)
- Официальный плагин **ModelContextProtocol** в движке: `ModelContextProtocol.GenerateClientConfig` → конфиги для Claude Code. Запуск из корня проекта.
- Воркфлоу: Claude Code = MCP-клиент рядом с редактором UE; параллельно спавнит тест-актёров, гоняет PIE, смотрит скрин. Для «крутить фил» — золото.
- Сторонние: UnrealClaude (захват вьюпорта — Claude видит себя), StraySpark (370+ тулзов, PIE), VibeUE.
- ⚠️ **БЕЗОПАСНОСТЬ: allowlist обязателен** (читать — да, удалять ассет — нет). **Auto-approve ВЫКЛ** до safe-профиля.
