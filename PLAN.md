## 0. О ЧЁМ ИГРА

**В двух словах (для друзей):** несёшься через неоновый киберпанк-город на бешеной скорости — бежишь по стенам, прыгаешь между крышами, скользишь под препятствиями. Чем чище проходишь — тем быстрее летишь. Один забег = несколько секунд чистого потока и адреналина. Потом сравниваешь время с друзьями и всем миром: рядом бежит призрак чемпиона, и ты пытаешься его обогнать. Жанр одной строкой: **скоростной паркур-рейсер от 3 лица — будто Ghostrunner встретил Trackmania.** Игра про скорость, точность и желание пройти трассу ещё чище.

Скоростная паркур-гонка от 3 лица: максимальная скорость, цена ошибки в долю секунды, накопление импульса за чистоту прохождения. Соревнование через призраков и лидерборды на сложных трассах — авторских и генерируемых. Киберпанк.
Референсы: Ghostrunner (поток), Warframe (импульс), Trackmania (соревнование/дейлики), Neon White (скорость+лидерборды), Titanfall 2 (фил).

## 1. ПОРЯДОК РАБОТЫ (железно)
80% успеха = game feel. Проверяется первым, на серой болванке, ДО всего.
1. Фил — манекен на gym-полигоне (см. §1-бис) + камера.
2. Одна ручная трасса + таймер + рестарт + свой призрак. Дать друзьям.
3. Призраки + лидерборд (асинхрон).
4. Правила трасс → модульная библиотека → дейлики (генерация).
5. Тюнинг статов (опц., отдельный режим).
6. Realtime-сеть — в самом конце, если вообще нужна.
Каждый этап дешевле провалить, чем следующий.

## 1-БИС. GYM-ПОЛИГОН ДЛЯ ФИЛА (18.06.2026)
Этап 1 — НЕ пустая коробка, а минимальный серый полигон: по одному эталонному элементу на каждую механику мувсета. Иначе нечего тюнить (вал-ран без стены не настроить, слайд без горки — тоже).
Элементы (серые блоки, без графики):
- **Пропасть** (провал) → дальность прыжка, рывок, coyote time, jump buffer.
- **Стена сбоку** → вал-ран (горизонтальный бег по стене).
- **Вертикальная стена / уступ** → вскарабкивание (climb / mantle через край).
- **Горка-рампа** → слайд-скат + набор импульса вниз.
- **Серия платформ** → чейнинг прыжков, сохранение импульса между приёмами.
Эти эталонные элементы — первые кирпичи модульной библиотеки §6 (gym перерастает в стройматериал, не выкидывается).
**Мувсет (дополнен):** бег, прыжок, рывок (КД), вал-ран, слайд, **вскарабкивание/мантл**.
**Рывок — по КД (решено), и это намеренно тактический ресурс.** КД делает ценным не сам рывок, а ТАЙМИНГ его траты. Следствия (источник глубины маршрутов без ручного дизайна):
- Тайминг важнее дистанции: путь чуть длиннее выгоден, если ставит рывок в идеальную точку.
- Комбо-ценность: рывок в слайд / рывок с горки даёт куда больше скорости, чем рывок «в пустоту» → линия строится вокруг таких связок.
- Цена ошибки: растратил рывок не вовремя → ждёшь КД, теряешь темп. Прямой путь с пустым рывком проигрывает кривому с эффективным.
Так развилки риск/награда (§5) рождаются сами, из взаимодействия КД и геометрии — игрок считает тайминги.

## 1-ТЕР. ПЕРВЫЙ ПЛЕЙБЛ — вертикальный срез для друзей (решено 18.06.2026)
Минимум, чтобы фил читался, и ни байта больше. Цель — честный ответ «тянет ли перепройти». Это шаг 2 из §1.
Что входит:
- **Одна трасса** из gym-элементов (§1-бис): пропасть → вал-ран → мантл → спуск-слайд → финиш. Серые блоки, без графики.
- **Спуск — кульминация в конце.** Раз спуск наш фокус, трасса ведёт к эпичному скоростному спуску, чтобы забег заканчивался на пике ощущения.
- **Таймер + мгновенный рестарт** (одна кнопка, без меню — критично для «ещё разок»).
- **Свой призрак — со 2-го заезда, ТОЛЬКО лучший (PB).** Один призрак = личный рекорд, перезаписывается при побитии. Не каждый заезд и не пачка — один эталон, твой потолок. Первый заезд его создаёт.
- **Спидометр** (приборка §3.3-кватер) — виден разгон.
Что НЕ входит: лидерборд, сеть, графика, вторая трасса, звук кроме ветра.
Суть: маленький срез, но содержит всю петлю «беги → ошибись/спасись → финишируй быстрее → ещё разок». Друг за 5 минут понимает игру.

## 2. КАМЕРА — 3 ЛИЦО (решено)
Видно персонажа (косметика + читаемость трассы), лучше пространственная осведомлённость, драйв через тело. Прецеденты: Sonic, Bomb Rush Cyberfunk, Solar Ash, Severed Steel, Pseudoregalia.
Камера = половина фила: слежение на скорости/поворотах, поведение при падении, не врезаться в стены, FOV-буст + наклон.
⚠️ На болванке попробовать обе камеры, решить руками.

## 3. GAME FEEL — ОБЯЗАТЕЛЬНОЕ
Суть: расширять окна по времени/позиции, чтобы игре подыгрывать игроку.
3.1 Базовый фил:
- Coyote time — прыжок ещё ~5 кадров после схода с края.
- Jump buffering — нажатие чуть раньше земли срабатывает в кадр касания.
- Corner correction — задел угол → подталкивает мимо, не клинит.
- Halved-gravity jump peak — на вершине прыжка гравитация вдвое слабее.
3.2 Камера/обратная связь (продают скорость):
- FOV растёт со скоростью (главный трюк).
- Наклон камеры на вал-ране/поворотах.
- Поджатие ног / руки в кадре на рывке/слайде.
- Звук + лёгкий motion blur на пике.
- Микро-тряска на приземлении/рывке.
3.3 Ядро — накопление скорости:
- Momentum preservation — импульс не сбрасывается между движениями.
- Накопление за чистоту — дольше без столкновений → быстрее. Видимый индикатор.
- Сброс при ошибке — врезался = потерял разгон.
- Momentum storage — прыжок с движущейся платформы добавляет её импульс.
- Мувсет: рывок (КД), вал-ран, слайд с поджатием, прыжок, вскарабкивание/мантл.

3.3-бис ЗАКОН ВЫСОТЫ = СКОРОСТИ (решено 18.06.2026): сохранение энергии, гравитация — друг на спуске.
- **Вниз → разгон.** Слайд по склону вниз копит скорость. Вал-ран по НИСХОДЯЩЕЙ стене — разгоняет (не просто держит).
- **Горизонт → сохранение.** Ровный вал-ран / бег держит импульс, не растит.
- **Вверх → плата.** Вскарабкивание, подъём, восходящий вал-ран — тратят накопленное.
Следствие для левелдизайна: трасса = управление высотой. Забрался наверх (дорого, медленно) → конвертируешь в скорость на спуске. Цикл «заберись — слети». Высота = запасённая скорость; спидранер думает «где набрать высоту, чтобы слететь быстрее», не «где короче». Рефы: SSX, Tony Hawk, Dying Light, реальный паркур.
⚠️ Гравитация-через-импульс FPS-зависима (RESEARCH §2 — главный риск честности) → спуски разгоняли бы по-разному на разном железе. Лечится ФИКС-ТАЙМСТЕПОМ (уже приоритет №1). Цена фичи = дисциплина по детерминизму.

3.3-тер СПАСЕНИЕ ПО ТАЙМИНГУ — wall-tech / kick-off (решено 18.06.2026). Столкновение НЕ бинарный провал, а лестница опций под уровень игрока (парирование как в Sekiro, для геометрии; рефы Titanfall, Mirror's Edge). 4 исхода от safe к pro:
1. **Объезд** — не берёшь рисковую линию. Медленнее, но цел. Доступно всем.
2. **Блок (ЗАЖАЛ кнопку)** — теряешь ЧАСТЬ импульса, но не всё. Тайминг не нужен, просто среагировал. Подушка для новичка.
3. **Парирование (попал в УЗКОЕ окно)** — сохранил импульс ПОЛНОСТЬЮ, лучше — перенаправил (отскок вперёд/вверх) + лёгкий буст. Про-игрок СПЕЦИАЛЬНО режет углы вплотную ради ускорения.
4. **Ничего / лоб на полной** → полный сброс импульса + микростан. Цена настоящей ошибки.
Правила: (а) АНТИСПАМ — окно парирования узкое, привязано к контакту; зажать заранее = только блок (п.2), не парир. (б) ЧЕСТНОСТЬ — окно одинаково на любом FPS → опять ФИКС-ТАЙМСТЕП.
Порог входа (РЕШЕНО — Trackmania-модель): темпом НЕ давим, безопасный путь всегда есть. Скорость нужна ради ТАБЛИЦЫ, не ради «пройти вообще». Один контакт читается 4 способами под 4 уровня игрока — никого не выгоняем (скилл-срез §5-бис). Отскок вверх → высота → скорость (кормит 3.3-бис).
3.3-кватер ОБРАТНАЯ СВЯЗЬ СКОРОСТИ — спидометр + сенсорика (решено 18.06.2026). Два РАЗНЫХ слоя, нужны оба, не подменять:
- **Спидометр = информация (мозг).** Для спидран-игры must-have: точное число → оптимизация линии, «здесь потерял 3 м/с». Рефы: Trackmania, Distance, bhop-ups.
- **Сенсорика = ощущение (тело).** FOV-кик + **ветер** (свист: громкость и тон растут со скоростью — главный дешёвый аудио-сигнал разгона; визуально — стрик-линии/частицы мимо камеры, искажение по краям экрана) + тряска + motion blur на пике. Без неё спуск вял даже при растущей цифре.
- ⚠️ Ловушка: только спидометр → игрок втыкает в цифру, не в мир, скорость не ЧУВСТВУЕТСЯ. Спидометр ДОПОЛНЯЕТ сенсорику, не заменяет.
- **Приборка спидранера:** спидометр + индикатор импульса/чистоты + таймер + сплиты. ОПЦИОНАЛЬНА/настраиваема (трейлер и новичок — спрятать, чистый кадр à la Mirror's Edge; оптимизатор — развернуть). Показывает горизонтальную скорость (вертикаль — решить на болванке).
- **Бонус:** спидометр обучает невидимым системам — видно, как цифра растёт на спуске / падает на подъёме (3.3-бис), как импульс капает за чистоту. Делает наши законы считываемыми + доверие к системе.

3.4 Философия: скорость ЗАРАБОТАНА, не выдана (Tribes, Quake, Titanfall, Ultrakill). Neon White: «игра должна научить тебя быть быстрым».

3.4-бис МУЗЫКА И ЗВУК (решено 18.06.2026).
- **Тон: жёсткая электроника — darksynth / drum‑n‑bass / industrial.** Агрессивно, качает, киберпанк. Школа Ghostrunner (Daniel Deluxe, Dan Terminus, GosT). НЕ эмбиент Mirror's Edge — у нас движуха.
- **ADAPTIVE, не ритм‑игра:** музыка РЕАГИРУЕТ на игрока, не наоборот. Слои (stems) нарастают с импульсом/скоростью — на чистом разгоне трек расцветает, на смерти/ошибке проседает (теряет слой). Вертикальное наслоение (FMOD/Wwise). Ритм‑игру (попадать в бит, Hi‑Fi Rush) НЕ берём — убьёт свободу линий и честность.
- **Музыка = второй индикатор импульса** (ушами): расцвела = в потоке; оборвалась = облажался. Аудио‑награда за чистоту, дублирует ядро 3.3. На эпичном спуске трек на полную — продаёт кульминацию.
- 🚨 **Музыка клиентская, на детерминизм НЕ влияет** (поверх симуляции, не трогает физику/тайминги).
- 🚨 **Болванку тестим БЕЗ музыки** (тишина + ветер + шаги). Музыка маскирует кривой фил → кажется кайфово из‑за трека, а движение сырое. Фил вкусный сам по себе в тишине, музыка ПОСЛЕ. Правило.
- **Продакшн:** Рамиль спросит знакомого музыканта (composer‑партнёр). На болванке музыки нет; темп‑треки только для прототипа трасс. Не бюджетный блокер.

3.5 НЕ делать на болванке: графику, трассы, Lumen/Nanite, бой, меню, тюнинг, модель, МУЗЫКУ (фил — в тишине).

## 4. ФИЗИКА И АРХИТЕКТУРА
Физику не планируют — настраивают итерациями. Фил живёт в пальцах. Планируешь возможность БЫСТРО крутить:
- Все параметры наружу (рывок/откат/гравитация/эйр-контроль) — менять в редакторе на лету (BP/DataAsset).
- Числа независимы.
- Быстрый цикл правка→проверка — главный фактор скорости.
- ⚠️ Не делать «идеальную песочницу» месяц. Достаточно удобную за пару дней.
4.1 КРИТИЧНО: фиксированный таймстеп физики (не к фрейму). Иначе на разном FPS разная скорость → лидерборды нечестные. UE: DeltaTime + фиксированный тик. Заложить сразу.

## 5. ЛЕВЕЛДИЗАЙН ТРАСС

**РЕШЕНО (18.06.2026): ЧАСТЬ 1 = ЗАКРЫТЫЕ ТРАССЫ-УРОВНИ, НЕ открытый мир.**
Почему уровни: (1) можно выкатывать контент частями, по карте за раз; (2) каждую карту дорабатываешь персонально, доводишь до кайфа; (3) на закрытых трассах отрабатываются основы — все великие гонки начинали так. Цель части 1 — пара трасс, по которым реально интересно бегать.
**Путь роста контента — лестница, не прыжок: ТРАССА → РАЙОН → ГОРОД.**
Когда накопится несколько трасс в одном сеттинге — сшить их в РАЙОН: общее бегабельное пространство, старые трассы = маршруты внутри него + связки между ними. Это мини-открытый мир контролируемого размера. Районы стыкуются → постепенно вырастает город. Каждый район — отдельная порция контента (катим частями), риск нарастает плавно, переиспользуются те же модули §6. Город = много сшитых районов.
**Открытый город — видение ЧАСТИ 2** (если доживём): трасса = A→Б по якорям внутри города, поиск маршрута на игроках. Модель — Mirror's Edge Catalyst. Технически реально на World Partition (UE 5.8), переиспользует модульную библиотеку §6. Сейчас НЕ строим — сложность мира убивает отработку основ.

5.1 «Широкий коридор»: явное направление А→Б, внутри линии риск/награда:
- Безопасная — длиннее, прощает, для новичка.
- Быстрая — короче, нужен точный вал-ран/рывок/импульс.
- Безумная — для задротов, 1 из 100, потолок скилла.
Ключ: линии дают РАЗНОЕ время. Скилл = выбор линии + чистота.
5.2 НЕ полная песочница — свобода ломает «правильную линию» → лидерборд разваливается.
5.3 СМЕРТЬ И ФЕЙЛ-СТЕЙТЫ (решено 18.06.2026). Без смерти у риска нет цены: ошибка = только потеря скорости → рисковая линия «бесплатна». Смерть-зоны дают риску настоящие зубы («цена ошибки в долю секунды»). Два типа фейла:
- **Мягкий — потеря импульса.** Лоб в стену / промах wall-tech (§3.3-тер) → сброс скорости + микростан. Не смерть, бежишь дальше медленно.
- **Жёсткий — смерть.** Улетел в пропасть / смертельную зону (киберпанк: обрыв, лазеры, провал) → МГНОВЕННЫЙ рестарт.
Правила: (а) смерть ДЕШЁВАЯ и мгновенная — рестарт по той же кнопке, без загрузок и наказания (цикл «ещё разок» священен; Neon White, Trackmania). (б) смерть-зоны — атрибут РИСКОВЫХ линий, НЕ основного пути: безопасная линия (Trackmania-порог входа §3.3-тер) без пропастей под ногами, медленно но не убивает. Хочешь срезать над бездной по безумной линии — там и ставка: влетел красиво или сдох.

## 6. DATA-DRIVEN ПАЙПЛАЙН ТРАСС
1. Фил → цифры (дальность рывка, высота прыжка, скорость вал-рана, импульс).
2. Правила/размеры → из цифр (мин/макс разрыв, длина стены, высота). «Словарь» левелдизайна.
3. Пресеты комбо-трюков → вылизанные модули, гарантированно проходимы.
4. Динамическая генерация → из проверенных модулей (WFC, не рандом).
5. Текстуры → последний слой.
6.1 Шаг 2 — это КОД: правила в данные = единый источник истины (ручная сборка + генератор + валидация). Поменял рывок → всё пересчиталось.
6.2 ⚠️ Генератор не раньше времени: сначала 10–15 модулей руками на реальном филе.
6.3 Процедурка vs честность: сид-генерация (у всех одна daily-трасса) / модульная сборка / процедурка как инструмент дизайнера.

## 7. ДВА РЕЖИМА КОНТЕНТА
- Curated (авторские, убер-сложные) — мастерство, киберспорт, стримеры. Ядро/лицо.
- Динамические (сид/daily) — свежесть, удержание, равные условия. Ретеншн-движок.
Один фундамент: модульная библиотека кормит оба.

## 8. СОРЕВНОВАНИЕ — АСИНХРОН, НЕ REALTIME
Призраки + лидерборды + дейлик по сиду.
- Все успешные (Trackmania, Neon White, Distance) — через призраков.
- Realtime-неткод на скорости = смерть для малых команд (Lemnis Gate закрылся, Distance отменил онлайн).
- Призраки = ~90% драйва за ~10% сложности, всегда доступны.
Опц. «live» = призраки-оверлей без коллизии.

## 9. ТЮНИНГ СТАТОВ (Forza-style, опц., рискованный)
НЕ прокачка, а тюнинг: общий бюджет одинаков, перераспределяешь внутри. Один робот, разные сетапы. Честно.
- Модули = сайдгрейды (меняют КАК двигаешься, не «быстрее»).
- Раздельные лидерборды: Spec (равны, киберспорт) + Open.
- Косметика без потолка, статы — нет.
- Потолок скорости от исполнения, не от модуля.
⚠️ В FPS-мувменте тюнинга почти нет. Вводить поздно, за флагом. Kill-switch: solved meta за недели → увести в casual.

## 10. ВЕРТИКАЛЬНОСТЬ И ПАДЕНИЯ

## 11. ПРОИЗВОДИТЕЛЬНОСТЬ (FPS на длинных трассах)

## 21. РЕШЕНИЯ ПОСЛЕ РЕСЁРЧА (18.06.2026) → детали в RESEARCH-movement.md
21.1 **Движение — развилка решена:** этап 1 (фил) делаем на **CMC‑subclass** (быстрый старт, готовые референсы — GitHub ExtendedCharacterMovement). Mover 2.0 (Beta в 5.8, детерминирован) — кандидат на потом. Решение «остаться на CMC+обёртка детерминизма / мигрировать на Mover» принимаем ФАКТОМ после этапа 1, когда фил найден и подходят лидерборды. Причина: переписать с готовым эталоном (цифры+ощущение) в разы проще, чем искать фил вслепую на сыром Mover; CMC = рабочий fallback. Фил‑цифры в DataAsset → переносятся куда угодно.
21.2 **§13 MCP — поправка:** официальный плагин UE 5.8 (ModelContextProtocol, Experimental) **НЕ умеет PIE/скриншоты/консоль из коробки** — только спавн актёров/материалы/Slate/автотесты. Чтобы Claude «играл уровень → смотрел → крутил фил» — нужен СТОРОННИЙ сервер (StraySpark с PIE / UnrealClaude с захватом вьюпорта) ИЛИ свой ToolsetDefinition (PIE/скрин/CVar).
21.3 **§8 призраки — уточнение:** записывать **transform/позу + интерполяция** (НЕ инпут — хрупко из-за FP-дрейфа). Старт с бесплатного **Ghost Replay System [Free for UE5]** (авг 2025). Инпут-запись держать только как серверную валидацию/античит. Лидерборды валидировать серверно.
21.4 **§4.1 подтверждено:** CMC AddForce/импульс‑с‑платформ ЗАВИСЯТ от FPS → дуги прыжков разные на железе = нечестные лидерборды. Лечить фикс‑таймстепом (manual accumulator вокруг CMC / Mover async). Smoothed Frame Rate Range (НЕ Use Fixed Frame Rate). Полный детерминизм недостижим — цель «достаточно честно» + серверная валидация.
21.5 **Тюнинг‑стенд (как строить):** params в DataAsset + UDeveloperSettings backed by CVars (live‑правка, INI‑персист) + Console Variables Editor (пресеты) + CSV Profiler/Visual Logger (метрики прогона). ## 5-БИС. ЛЕВЕЛДИЗАЙН — ПЕРЕОСМЫСЛЕНО (18.06.2026): ЧЕКПОИНТЫ-ЯКОРЯ + СВОБОДА МАРШРУТА
Корректирует §5. Игра = площадка для спидранов → emergent-маршруты это ФИЧА, не баг.
5б.1 **Два вида срезок (различать жёстко):**
- ЧИТ-срез = эксплойт коллизии / физ-баг / телепорт сквозь геометрию → УБИРАЕМ.
- СКИЛЛ-срез = через механики (накопил импульс, тормознул ради разгона, точный безумный прыжок, хитрый маршрут) → **СВЯТОЕ, не трогаем.** Душа спидрана.
5б.2 **Чекпоинты = ЯКОРЯ, не рельсы:** говорят «здесь надо реально побывать на скорости» (против читов мимо трассы), но НЕ говорят КАК их собрать. Между якорями — полная свобода маршрута. Игрок сам планирует: где тормознуть для разгона, откуда прыгнуть, как срезать импульсом.
5б.3 **3 линии (безопасная/быстрая/безумная) — ПОДСКАЗКИ-ориентиры, не рельсы.** Мастер найдёт 4-ю линию, которую не задумывали — это победа дизайна, не провал. Skill ceiling бесконечный.
5б.4 **Философия «не патчить emergent»** (Quake/Titanfall/Ghostrunner): bunny hop, rocket jump родились багами → стали ядром жанров. Патчить ТОЛЬКО то, что ломает физику насквозь; неожиданные маршруты через механики — оставлять.
5б.5 **Искусство дизайна якорей:** ставить чекпоинты так, чтобы их нельзя собрать «не по духу» (через дыру коллизии), но маршрут между ними максимально свободен. Узкие якоря, широкая свобода.
5б.6 **Баланс линий доказывать ЛОГАМИ** (время/линия, кто что выбирает), не интуицией — тот же логгер, что в тюнинг-стенде §22. Читаемость развилок на скорости (визуальный язык: цвет/форма/свет). Чекпоинт = 3 функции: валидация заезда + splits таймера + точка рестарта practice (§23).

## 24. РЕПЛЕИ, ШЕРИНГ И «ТЕАТР ПОВТОРОВ» (решено 18.06.2026)
База — transform/поза‑реплей (§21.3), лёгкий переносимый файл.
24.1 **Бэкенд лидербордов:** старт на **Steam Leaderboards + UGC‑призраки**, рекорд без смотрибельного реплея не засчитывается (честность через верификацию реплеев + модерация, как speedrun.com). Свой сервер — потом и только при упоре в то, чего Steam не даёт (глобальные синхронные daily, авто‑античит, веб‑таблицы). EOS не нужен (PC‑only).
24.2 **Проигрыватель реплея (scrubbing):** таймлайн‑бегунок (перемотка/пауза/замедление) + СВОБОДНАЯ камера (смотреть заезд изнутри движка с любого ракурса). Это сильнее видео — встроенный «театр повторов» для анализа/обучения/стримеров. Формат файла закладывать под scrubbing (таймстеп‑сэмплы + индекс по времени).
24.2-бис **РЕЖИМ РАЗБОРА ЗАБЕГА — «где проебался» (решено 18.06.2026).** Аналитический слой поверх скраба, почти бесплатный — данные уже пишутся (призрак = transform‑реплей). Состав:
- Скраб + замедление + покадровка + свободная камера (из 24.2).
- **График скорости на таймлайне** — видно визуально, где спидометр просел = где потерял темп.
- **Маркеры потери импульса** — точки на трассе, где сработал мягкий фейл (стена / промах wall‑tech §3.3‑тер). Подсвечено «здесь проебался».
- **Дельта к PB‑призраку** — наложить текущий заезд на лучший бок о бок: где отстал, где выиграл (Trackmania media tracker).
Зачем: «проиграл и не понял почему» → «вижу, на спуске недокрутил». Ретеншн + самообучение. Scope: НЕ в первый плейбл (§1‑тер минимум), появляется со слоем реплеев/призраков — но технически бонус поверх уже записываемого призрака.
24.3 **Шеринг забегов комьюнити (виральность + ретеншн):**
- Прямой файл `.replay` (скинул другу → загрузил → бежит против твоего призрака).
- Steam Workshop (публикуй/подписывайся на заезды).
- Автоматом из лидерборда (топ‑N призраков подтягиваются на трассе через UGC).
24.4 **Формат файла реплея — заложить сразу:** id+версия трассы (валиден только для своей версии; иначе помечается несовместимым, не ломает забег); метаданные (ник/время/дата/сетап управления); квантование позы (лёгкость для мгновенного шеринга).
